## 一、项目初始化（T1）

### 做了什么
用 `uv init --package --name repopilot --python 3.10` 创建项目骨架，然后手写 pyproject.toml。

### pyproject.toml 关键配置
```toml
[project.scripts]
repopilot = "repopilot.cli:app"   # 安装后命令行输入 repopilot 就调用 cli.py 里的 app
```
这是Python CLI的标准做法：setuptools的entry_points，pip install -e .后repopilot命令就可用。

### CLI框架为什么选Typer不选Click/Argparse
- Argparse是标准库但太原始，嵌套命令写起来很麻烦
- Click强大但需要手写很多样板代码
- Typer基于Click，但用Python类型注解自动生成参数解析，代码量减少60%
- 例：`def main(repo: str = Option(".", "--repo"))` 自动生成 `--repo` 参数

### 测试框架为什么选pytest
- 不用写class继承unittest.TestCase
- 直接用assert就行，不需要self.assertEqual
- fixture系统强大
- 插件丰富（pytest-timeout、pytest-cov）

## 二、配置系统（T2）

### 做了什么
实现了三层配置优先级：默认值 < config.toml < 环境变量 < 命令行参数。

### 为什么用TOML不是YAML/JSON
- TOML是Python社区标准（pyproject.toml就是TOML）
- 比JSON好写（支持注释）
- 比YAML简单（没有 Norway problem: `NO` 被解析成false）
- Python 3.11自带tomllib，3.10用tomli backport

### 熔断器实现（Circuit Breaker）
三个状态：CLOSED（正常）、OPEN（熔断）、HALF_OPEN（半开试探）。

```python
class CircuitBreaker:
    def __init__(self, threshold=5, recovery_timeout=30):
        self.failure_count = 0
        self.threshold = threshold         # 连续失败N次就熔断
        self.recovery_timeout = recovery_timeout  # 熔断30秒后半开
        self.state = "CLOSED"
        self.last_failure_time = 0

    def call(self, func, *args, **kwargs):
        if self.state == "OPEN":
            if time.time() - self.last_failure_time > self.recovery_timeout:
                self.state = "HALF_OPEN"    # 半开，放一个请求试探
            else:
                raise CircuitOpenError("Circuit breaker is open")
        try:
            result = func(*args, **kwargs)
            self.on_success()              # 成功→重置计数
            return result
        except Exception as e:
            self.on_failure(e)             # 失败→计数+1
            raise

    def on_success(self):
        self.failure_count = 0
        self.state = "CLOSED"

    def on_failure(self, exc):
        self.failure_count += 1
        self.last_failure_time = time.time()
        if self.failure_count >= self.threshold:
            self.state = "OPEN"
```

### 可重试错误分类
只重试能恢复的错误，不重试认证错误：
```python
def _is_retryable(exc):
    if isinstance(exc, (openai.APITimeoutError,
                         openai.APIConnectionError,
                         openai.RateLimitError)):
        return True          # 超时/连接错/限流，重试可能成功
    if isinstance(exc, openai.BadRequestError):
        return False         # 400错误，参数错了重试也没用
    if isinstance(exc, openai.AuthenticationError):
        return False         # key错了，重试没用
    if "timeout" in type(exc).__name__.lower():
        return True
    return False
```

### 指数退避重试

## 四、沙箱系统（T4+T5）

### Sandbox抽象基类
所有工具只依赖Sandbox抽象类，不关心是Local还是Docker，这是依赖注入。

```python
class Sandbox(ABC):
    @abstractmethod
    def setup(self): ...       # Local:检查目录存在; Docker:启动容器
    @abstractmethod
    def teardown(self): ...    # Local:没事; Docker:停止删除容器
    @abstractmethod
    def read_file(self, path, offset=0, limit=200): ...
    @abstractmethod
    def write_file(self, path, content): ...
    @abstractmethod
    def edit_file(self, path, old_string, new_string): ...
    @abstractmethod
    def exec(self, command, timeout=120, cwd=None): ...
    @abstractmethod
    def glob(self, pattern): ...
    @abstractmethod
    def grep(self, pattern, glob_filter=None, ignore_case=False): ...
    @abstractmethod
    def list_dir(self, path=".", max_depth=2): ...

    # __enter__/__exit__支持with语句
    def __enter__(self):
        self.setup()
        return self
    def __exit__(self, *exc):
        self.teardown()
```

### 路径安全（_safe_path）——防穿越攻击
这是安全关键代码：

```python
def _safe_path(self, user_path):
    p = (self.repo_path / user_path).resolve()   # resolve()处理../和符号链接
    try:
        p.relative_to(self.repo_path)            # 确认路径在repo内
    except ValueError:
        raise PermissionError(
            f"Path escapes repo: {user_path} -> {p}"
        )
    return p
```
**举个攻击例子**：用户请求`read_file("../../etc/passwd")`
- `repo_path / "../../etc/passwd"` = `/home/user/project/../../etc/passwd` = `/home/user/etc/passwd`（不对）
- `.resolve()` = `/etc/passwd`（真实路径）
- `.relative_to("/home/user/project")` → 抛出ValueError → PermissionError阻止

### LocalSandbox.exec实现
```python
def exec(self, command, timeout=120, cwd=None):
    workdir = str(self.repo_path)
    if cwd:
        workdir = str(self._safe_path(cwd))
    try:
        result = subprocess.run(
            command,
            shell=True,                 # Windows需要shell=True才能用内置命令
            cwd=workdir,
            capture_output=True,
            text=True,
            timeout=timeout,
            encoding="utf-8",
            errors="replace",           # 遇到非法字符不崩
        )
        return ExecResult(
            command=command,
            stdout=result.stdout or "",
            stderr=result.stderr or "",
            exit_code=result.returncode,
            timed_out=False,
        )
    except subprocess.TimeoutExpired as e:
        return ExecResult(
            command=command,
            stdout=(e.stdout or b"").decode("utf-8", errors="replace"),
            stderr=(e.stderr or b"").decode("utf-8", errors="replace"),
            exit_code=-1,
            timed_out=True,
        )
```

### 原子写入（防止半写入损坏文件）
```python
def write_file(self, path, content):
    p = self._safe_path(path)
    p.parent.mkdir(parents=True, exist_ok=True)
    # 先写临时文件，再os.replace原子替换
    fd, tmp = tempfile.mkstemp(dir=str(p.parent), prefix=f".{p.name}.tmp.")
    try:
        with os.fdopen(fd, "w", encoding="utf-8") as f:
            f.write(content)
        os.replace(tmp, str(p))   # 原子操作：要么完全替换，要么不变
    except:
        os.unlink(tmp)            # 清理临时文件
        raise
```
为什么不直接写？如果写一半断电，文件就损坏了（半写）。临时文件+rename保证原子性。

### Docker Sandbox实现关键点
- 用Docker SDK: `docker.from_env().containers.run()`
- 挂载: `volumes={str(repo_path): {'bind': '/workspace', 'mode': 'rw'}}`
- 资源限制: `mem_limit="2g", cpu_quota=200000`（cgroups，2核）
- 网络: `network_mode="bridge"`（联网）或`network_disabled=True`（断网）
- 命令注入防护: 用shlex.quote()转义参数，防止`; rm -rf /`注入
- exec: `container.exec_run(cmd, workdir="/workspace")`

## 五、权限引擎（T6）

### 四种模式（面试必背）

| 模式 | 读 | 写 | 执行 | 场景 |
|------|----|----|------|------|
| auto | ✅ | ✅ | ✅ | 自己的项目，快速迭代 |
| confirm | ✅ | 询问 | 询问 | 陌生代码/生产环境 |
| edit-only | ✅ | ✅ | ❌ | 不想让它执行命令 |
| deny | ✅ | ❌ | ❌ | 纯代码审查 |

### 工具分Tier
```python
TIER_READONLY = "readonly"     # read_file, grep, glob, list_dir, get_repo_tree
TIER_WRITE = "write"           # write_file, edit_file
TIER_EXEC = "exec"             # bash, run_python
```
只读工具总是允许，写和执行根据模式决定。

### 黑名单（patterns.py）
内置危险命令正则，任何模式下都拒绝：
```python
DANGEROUS_PATTERNS = [
    r"rm\s+-rf?\s+[/~]",       # rm -rf /, rm -rf ~
    r"del\s+/[SQ]",            # del /S /Q 系统盘
    r"curl\s+.*\|\s*(sh|bash)", # curl|sh
    r"wget\s+.*\|\s*(sh|bash)",
    r"sudo\s+",                # sudo
    r"su\s+(-|root)",          # su root
    r"chmod\s+777",            # chmod 777
    r">\s*~?\.ssh/",           # 写入.ssh
    r"git\s+push\s+.*(-f|--force)",  # force push
]
```

### 检查流程
```python
def check_tool(self, tool_name, args):
    # 1. 只读工具总是允许
    if tool_tier == TIER_READONLY:
        return PermissionDecision("allow")

    # 2. 检查黑名单（所有模式下都检查）
    if self._matches_blacklist(tool_name, args):
        return PermissionDecision("deny", "Blocked by safety rules")

    # 3. 根据模式决定
    if self.mode == "auto":
        return PermissionDecision("allow")
    elif self.mode == "deny":
        return PermissionDecision("deny", "Mode is deny")
    elif self.mode == "edit-only":
        if tool_tier == TIER_EXEC:
            return PermissionDecision("deny", "Exec blocked in edit-only mode")
        return PermissionDecision("allow")
    elif self.mode == "confirm":
        return PermissionDecision("ask", f"Allow {tool_name}?")
```

### ToolRegistry
```python
class ToolRegistry:
    def __init__(self, permission_engine=None):
        self._tools = {}
        self._permission = permission_engine

    def register(self, tool):
        self._tools[tool.name] = tool

    def schemas(self):
        """返回给LLM的OpenAI格式schemas"""
        return [
            {"type": "function", "function": {
                "name": t.name,
                "description": t.description,
                "parameters": t.parameters,
            }}
            for t in self._tools.values()
        ]

    def execute(self, name, args, sandbox, extra=None):
        # 1. 查工具
        tool = self._tools[name]
        # 2. 权限检查
        if self._permission:
            d = self._permission.check_tool(name, args)
            if d.action == "deny":
                return ToolResult(error=f"Permission denied: {d.reason}")
        # 3. 执行
        return tool.execute(args, sandbox, extra=extra)
```

### bash工具的Windows命令翻译
```python
def _windows_cmd_fix(cmd):
    # 链式命令不翻译（&&/||/;），避免破坏复杂命令
    if any(kw in cmd for kw in ["&&", "||", "|", ";"]):
        return cmd
    # python/pip/git/pytest/npm等跨平台工具不翻译
    if cmd.startswith(("python", "py", "pip", "git", "npm", "pytest", "uv", "node")):
        return cmd
    # 简单翻译
    translations = {"ls": "dir", "cat": "type", "pwd": "cd"}
    parts = cmd.split(None, 1)
    first = parts[0].lower()
    if first in translations:
        return translations[first] + " " + (parts[1] if len(parts)>1 else "")
    return cmd
```

```python
def build_messages(self):
    messages = [{"role": "system", "content": self.system_prompt}]

    # L0.5: 项目上下文（repo map + memory）
    project_parts = [self.repo_map_str]
    if self.memory_str:
        project_parts.append(f"## Memory\n{self.memory_str}")
```

```python
def tool_compact(text, max_head=2000, max_tail=5000):
    if len(text) <= max_head + max_tail + 50:

    return text[:max_head] + f"\n...[truncated {len(text)-max_head-max_tail} chars]...\n" + text[-max_tail:]
```

## 十、Agent Loop（T14）+ REPL

### 核心循环（run_agent）
```python
def run_agent(task, repo_path, llm, sandbox, ...):
    # 初始化：加载system prompt、注册工具、构建repo map、创建ContextManager
    ctx = ContextManager(budget_tokens, system_prompt, repo_map_str, memory_str)
    ctx.add_user(task)

    for step in range(max_steps):
        # 1. 检查压缩
        if ctx.needs_compaction() == "auto":
            ctx.compact("auto", llm)
        elif ctx.needs_compaction() == "micro":
            ctx.compact("micro", llm)

        # 2. 调用LLM
        resp = llm.chat(ctx.build_messages(), tools=tool_schemas, temperature=0.2)

        # 3. 解析回复
        parsed = parse_response(resp.content, resp.tool_calls)

        # 4. 处理工具调用
        for tc in parsed.tool_calls:
            result = registry.execute(tc.name, tc.arguments, sandbox)
            ctx.add_tool_result(tc.id, result.content, is_error=bool(result.error))

        # 5. 纯文本回复 = 任务完成
        if not parsed.tool_calls:
            ctx.add_assistant(parsed.content)
            return RunResult(status="completed", summary=parsed.content, steps=step, ...)

    return RunResult(status="max_steps", ...)  # 步数超限
```

### Response Parser
优先用OpenAI原生tool_calls；如果模型输出XML格式（`<tool_call>...</tool_call>`），用XML parser兜底解析。这是因为不是所有模型都支持function calling，或者有时模型会自己输出XML。

### REPL（持久多轮对话）
REPL和单次run_agent的关键区别：
- ContextManager在多轮间**保持存活**（不重建）
- 每轮用户输入后进入自己的子循环（有自己的sandbox per turn）
- Slash命令（/exit, /model, /clear, /memory等）不经过LLM，直接在REPL层处理
- /clear重建ContextManager（清空对话历史）
- /resume从SessionStore的JSONL回放历史事件重建ContextManager

```python
class ReplSession:
    def __init__(self, ...):
        self._build_context()   # 创建ContextManager，加载repo map和memory

    def run_turn(self, user_message):
        self.ctx.add_user(user_message)
        with self._make_sandbox() as sb:   # 每轮一个新sandbox
            while turn_steps < max_turn_steps:
                # ... 同agent loop ...

    def do_clear(self):
        self._build_context()   # 重建一切
        self.steps = 0
        self.cost_tracker = CostTracker()
```
