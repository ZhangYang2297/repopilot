# RepoPilot 项目开发指南

> Local-first Code Agent CLI, inspired by Claude Code and Codex CLI.

## 项目简介

RepoPilot 是一个本地优先的代码助手 CLI 工具。在终端 `cd` 到项目目录，输入 `repopilot` 进入交互模式，用自然语言告诉它要做什么——修复bug、添加功能、重构代码、运行测试——它会自动读代码、搜索、编辑、执行命令、验证结果。

- **纯ReAct架构**：单模型推理循环，不多agent编排，对齐Claude Code/Codex实现
- **Docker沙箱隔离**：代码执行在容器内，有资源限制、网络控制、路径安全
- **四层权限控制**：auto/confirm/edit-only/deny
- **分层记忆系统**：全局记忆→项目记忆→压缩摘要→近期消息→工具结果
- **无RAG/无向量库**：grep/glob/tree-sitter确定性检索，Markdown文件持久记忆
- **三档模型路由**：fast/default/strong，LiteLLM统一调用
- **生产级稳定性**：熔断器、指数退避、原子写入、路径穿越防护、超时控制

## 快速开始

### 安装
```bash
cd RepoPilot
pip install -e .
```

### 配置
首次运行自动启动配置向导：
```bash
repopilot
```
输入模型名（如`openai/doubao-seed-evolving`）、API Key、Base URL即可。

配置文件保存在 `~/.repopilot/config.toml`。

### 使用方式

**交互模式（推荐）：**
```bash
cd your-project
repopilot              # 进入REPL，当前目录作为项目根目录
```

**单次任务模式：**
```bash
repopilot chat "fix the failing test in test_auth.py"
repopilot chat "add a --verbose flag to the CLI" -r ./my-project
```

**指定沙箱/权限/模型：**
```bash
repopilot --sandbox docker --approval-mode confirm -m openai/gpt-4o
```

## REPL Slash 命令

| 命令 | 功能 |
|------|------|
| `/exit`, `/quit` | 退出 |
| `/help` | 显示帮助 |
| `/model [name]` | 查看或切换模型 |
| `/approval [mode]` | 切换权限模式 (auto/confirm/edit-only/deny) |
| `/compact` | 手动触发上下文压缩 |
| `/clear` | 清空对话 |
| `/cd [path]` | 切换工作目录 |
| `/memory [note]` | 查看记忆或追加笔记（加`--global`存全局） |
| `/resume [id]` | 恢复历史会话（默认最近） |
| `/sessions` | 列出最近会话 |
| `/cost` | 查看token用量和费用 |
| `/status` | 查看当前配置 |

## 项目记忆

在项目根目录创建 `REPOPILOT.md`，Agent每次启动会自动读取其中的指令：

```markdown
# Project Memory

## Build/Test Commands
- Test: python -m pytest tests/ -v
- Lint: ruff check .
- Run: python -m app

## Conventions
- Use type hints on all functions
- Follow existing code style
- Never modify files in migrations/
```

全局记忆在 `~/.repopilot/REPOPILOT.md`，所有项目生效。

记忆文件可以直接用编辑器修改，也可以在REPL中用 `/memory 注意要先跑测试` 追加。

## 运行评测

```bash
# E2E任务评测（12个任务）
python -m eval.run --tasks 1-12 --max-steps 80

# 记忆系统专项评测
python eval/test_memory_quick.py

# 单元测试
python -m pytest tests/ --tb=short -q
```

## 技术栈

| 组件 | 技术选择 | 为什么 |
|------|---------|-------|
| CLI框架 | Typer | 基于类型注解，代码最少 |
| 终端渲染 | Rich | Markdown/彩色/Spinner/表格 |
| LLM调用 | LiteLLM | 统一100+模型API |
| 代码解析 | tree-sitter | 增量AST解析，多语言 |
| 容器 | Docker SDK | 沙箱隔离 |
| 日志 | structlog | 结构化JSON日志 |
| 会话存储 | JSONL + SQLite | 追加写快+查询快 |
| 配置 | TOML (tomli/tomllib) | Python标准，支持注释 |
| 测试 | pytest + pytest-timeout | 简洁+超时控制 |
| 忽略规则 | pathspec | .gitignore兼容 |

## 不使用的技术（及原因）

| 技术 | 不使用原因 |
|------|-----------|
| LangChain/LangGraph | 太重，核心就是while循环不需要额外框架 |
| RAG/向量数据库/Embedding | 单租户CLI，grep/glob/tree-sitter确定性检索更精准 |
| 多Agent编排（Planner+Coder+Reviewer） | 强模型单ReAct足够，Claude Code/Codex也是单模型 |
| ChromaDB/FAISS | 不需要向量检索 |
| Async（asyncio）| CLI工具同步足够简单，async增加复杂度 |
